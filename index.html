The shadowDocument Magic

Vanilla SPA automatically injects shadowDocument into every script tag. It's your component's Shadow Root â€” no setup needed. 

<script>
// Direct Shadow DOM access
const button = shadowDocument.getElementById('my-btn');
const output = shadowDocument.querySelector('.result');

button.addEventListener('click', () => {
  output.textContent = 'Clicked!';
});

// Access the host element
const host = shadowDocument.host;
console.log(host.tagName); // "APP-START"
</script>
Note: Scripts run as ES modules, so no onclick="fn()" in templates. Use addEventListener or the data-action pattern instead.

Using Functions in Templates

Since scripts are modules, functions aren't global. The solution: data-action pattern + bindActions()

Create: /src/bindActions.js

export function bindActions(shadowDocument, actions) {
  shadowDocument.querySelectorAll('[data-action]').forEach(el => {
    const fn = actions[el.dataset.action];
    if (fn) {
      bus.addEventListener('click', fn, el);
      el.addEventListener('click', e => bus.dispatchEvent(e, el));
    }
  });
}

Use in component:

<template>
  <button data-action="greet">Say Hello</button>
  <button data-action="celebrate">Celebrate</button>
  <output id="out">â€“</output>
</template>

<script>
import { bindActions } from '/src/bindActions.js';

const out = shadowDocument.getElementById('out');

function greet() { out.value = 'Hello! ğŸ‘‹'; }
function celebrate() { out.value = 'Party! ğŸ‰'; }

Component Communication

Use @vanillaspa/event-bux for context-based events. Events are automatically cleaned up when components are removed.

Setup in src/main.js:

import '@vanillaspa/web-components';
import * as bus from '@vanillaspa/event-bus';
window.bus = bus;

In your component:

<script>
const host = shadowDocument.host;

// Fire an event
function notify() {
  bus.dispatchEvent(
    new CustomEvent('user-action', { 
      detail: { action: 'clicked' } 
    }),
    host  // Context: only this instance's listeners
  );
}

// Listen for events
bus.addEventListener('user-action', (e) => {
  console.log('Action:', e.detail.action);
}, host);
</script>

Offline Database (Optional)

Add @vanillaspa/sqlite-database for a full SQLite database in the browser. Persisted in OPFS, runs in Web Worker. 

npm install @vanillaspa/sqlite-database
<script>
import * as sqlite from '@vanillaspa/sqlite-database';

// Create database
await sqlite.createDB('my-app');

// Create table
await sqlite.executeQuery(`
  CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    name TEXT,
    email TEXT
  )
`, 'my-app');

// Insert data (SQL injection safe)
await sqlite.executeStatement({
  sql: 'INSERT INTO users (name, email) VALUES (?, ?)',
  values: ['Anna', 'anna@example.com'],
  name: 'my-app'
});

// Query
const users = await sqlite.executeQuery(
  'SELECT * FROM users',
  'my-app'
);
</script>
